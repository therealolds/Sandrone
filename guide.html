Sandrone – Tool Guide

Every tool runs entirely in your browser. Nothing is uploaded or stored on any server. This guide explains what each tool does, how to use it, and the less obvious features.

Data visualization

JSON Viewer

View, navigate, search and compare JSON files — with up to three documents side by side.

Open tool →

Loading JSON

Paste JSON directly into the text area, or pick a .json / .txt file with the file picker. Click Format to pretty-print the content (configure the indent width, 0–10 spaces, with the number field).

The text area has a line-number gutter on the left. Even and odd rows are shaded differently so you can track a line number at a glance. If the JSON is invalid, the offending line turns red in the gutter and an error banner appears below with the exact line, column, and parser message. Click ↳ Jump to line N in the banner to move the cursor straight there.

Memory slots (M1 / M2 / M3)

Three ephemeral slots hold JSON for the lifetime of the page (cleared when you close or reload).

ActionHow
SaveShift / Ctrl / Alt + click the slot button
LoadClick the slot button
RenameRight-click the slot button

The slot button label shows a truncated version of the custom name (if set) and a * when data is stored. The full stored content is shown in the button tooltip.

Interactive view

Click Interactive view to render the JSON as a collapsible tree. Each node can be expanded or collapsed with the + / - toggle. Click any line to see its full path in the Selected path panel on the right (e.g. root>glossary>GlossDiv>title>). During a search, the panel also shows the current match number and the matched value.

Multi-column view

The Visible: row at the top of the Interactive View card contains toggle buttons — Input, M1, M2, M3 — one per source. Memory slot buttons are enabled only when that slot contains data.

  • Click a button to add or remove that column. At least one column must always remain visible.
  • When multiple columns are shown, each has its own independent tree. Clicking a node in any column makes that column active — the path panel and the search pane on the right will then operate on it (a small label indicates which slot is active).
  • Expand All and Collapse All apply to every visible column at once.

Side panel

The sticky panel to the right of the tree has four sections:

  • Selected path — shows the full path of the last clicked node, and during a search, the current match's path, match index, type, and value.
  • Advanced search — described in detail below.
  • Utilities — helper tools for extracting data from the JSON (see below).
  • View tools — font size control and a Jump to the top button (see below).

Advanced search

The search panel operates on the active column's JSON. Text search and number search can be combined in a single query — only nodes matching both are returned.

FieldWhat it does
Text searchMatches any key or value that contains the term. Enable Case sensitive to make the match exact-case, or % to treat % as a wildcard (e.g. user%id matches user_id and userId). Use key: value syntax to match a specific key-value pair.
Limit search pathRestricts the search to a subtree. Paste a path copied from the Selected path panel (e.g. root>data>users>). The statistics (if enabled) are also scoped to this subtree.
Number searchFinds numeric values matching a condition. Set an optional key filter to restrict to values under a specific key name (e.g. version), choose an operator (=, ≠, >, <), and enter the target number.
Round (match rounded value)Before comparing, rounds the JSON value to the same number of decimal places as the input. For example, entering 1.23 matches 1.2345 but not 1.24.
Show statisticsWhen enabled, computes aggregate statistics over all matched numeric values and displays them below the toggles (see Statistics below).

Click Search (or press Enter in any search field) to run. Use Previous / Next to step through results. Each match is highlighted in the tree and its ancestors are expanded automatically.

Tip: Running a search while already in Interactive View does not reset the other columns — it only navigates within the active one.

Statistics

Tick Show statistics before (or after) running a number search. A panel expands showing the following values computed across all matched numbers:

StatDescription
nCount of matched numeric values.
SumTotal of all matched values.
MeanArithmetic average.
Std devPopulation standard deviation.
p50 (median)50th percentile — the middle value.
p9595th percentile.
p9999th percentile.

If Limit search path is set, statistics are scoped to the filtered results only. Toggling the checkbox updates the panel immediately without re-running the search.

Graph view

Renders the JSON structure as a node graph (SVG). Click any box to expand or collapse its children. Drag to pan. Use Expand All / Collapse All from the toolbar.

Font size and navigation

The View tools section of the side panel contains:

  • Font size — the 🔍 − / + control adjusts the tree font size between 8 px and 24 px. The current size is shown between the buttons.
  • Jump to the top — scrolls the page back to the top instantly.

Send to JSON Comparator

The → JSON Comparator button opens a modal. Choose a source for JSON A and JSON B — any memory slot, the current input, a file from disk, or a paste area — then click Open in Comparator →. The Comparator opens in a new tab with both sides already populated.

XML Viewer

View and analyze XML documents with formatted text, tree, table, and graph views.

Open tool →

Loading XML

Paste XML into the text area or load a file. Click Format to re-indent with the chosen indent size.

Views

  • Interactive view — collapsible tree, same layout as JSON Viewer including a Selected path panel and three memory slots.
  • Table view — hierarchical detail view, good for inspecting individual elements and attributes.
  • Graph view — SVG node graph, click to expand.

JSON Tracker

Import a browser HAR export and browse, filter and inspect every JSON response captured in the network traffic.

Open tool →

What is a HAR file?

A HAR (HTTP Archive) file is a JSON file that your browser's DevTools can export. It contains every network request and response — URL, method, status, headers, body, timing — exactly what you see in the Network panel.

Exporting from your browser

  1. Open DevTools (F12) and go to the Network tab.
  2. Interact with the page you want to inspect (load a page, click buttons, submit forms).
  3. Right-click any request in the list → Save All As HAR
    Firefox: Salva tutto come HAR  ·  Chrome: Save all as HAR with content
Tip: Enable "Persist logs" in DevTools before navigating so requests from previous pages are also captured.

Browsing entries

Drop the .har file onto the page or click the drop zone to browse. The tool defaults to showing only JSON responses; remove the filter to see all traffic.

The table shows method (color-coded), status code, URL (path only; full URL on hover), content type, response size, and response time. Click any row to preview the response in the right panel — JSON bodies are rendered as a collapsible tree; other bodies are shown as raw text. POST/PUT request bodies are shown too when present.

The header checkbox selects or deselects all currently visible rows.

Sending to other tools

Tick one or more rows with the checkboxes, then use the action bar that appears:

  • Open in JSON Viewer — up to 4 entries are sent: the first loads into the Input slot, the rest into M1 / M2 / M3. The viewer opens in a new tab with the data pre-loaded and Interactive View already open.
  • Send to Comparator — available only when exactly 2 entries are selected. Both response bodies are sent to the JSON Comparator in a new tab.

String preparation

String Formatter

Turn a list of lines into quoted, comma-separated values — useful for SQL IN clauses or R vectors.

Open tool →

Usage

Paste one value per line into the input area. Choose single or double quotes and an output layout (vertical — one value per line — or inline). Optionally wrap the whole result in parentheses. Click Format to generate the output.

Also includes a Line Padder utility: pad each line to a fixed width with a configurable fill character, useful for aligning columns in text output.

SQL Insert Generator

Generate SQL INSERT statements from a CSV file.

Open tool →

Usage

  1. Paste CSV text or load a .csv file. The delimiter is detected automatically (comma, semicolon, or tab) but can be changed manually.
  2. The first row is treated as column headers. A column grid is shown — tick the Quote checkbox for each column whose values should be wrapped in single quotes in the output.
  3. Enter the target table name and click Generate. The result is a multi-row INSERT INTO … VALUES … statement.

SQL Update Generator

Generate SQL UPDATE statements from a CSV file.

Open tool →

Usage

  1. Paste or load a CSV. The delimiter is auto-detected.
  2. In the column grid, assign each column a role:
    • WHERE — used in the WHERE condition (key columns).
    • UPDATE — used in the SET clause.
    • Ignore — excluded from the output.
  3. Toggle Quote per column to wrap values in single quotes.
  4. Enter the table name and click Generate.

Race Calendar JSON Generator

Build a structured race-event JSON from a CSV — intended for use with the RaceCalendar app.

Open tool →

Usage

Paste or load a CSV where each row is a race or a session. A MAIN column (value 1 or 0) marks whether the row is a main race event or a sub-session. Configure the delimiter, then click Generate to produce the JSON. The tool groups sessions under their parent race and warns about any unmatched rows. Supports Excel date serial number conversion.

File comparison

JSON Comparator

Diff two JSON documents — structurally or as raw text.

Open tool →

Modes

ModeBehavior
fullStructural diff; object keys and array elements are both sorted before comparison. Two arrays with the same items in different order are considered equal.
orderedStructural diff; object keys are sorted, but array order is preserved. Use this when sequence matters.
exactRaw text diff, line by line. No parsing — whitespace and key order matter.

Differences are shown with + for lines only in the second document and − for lines only in the first. Click Format to pretty-print both inputs before comparing.

Tip: The → JSON Comparator button in the JSON Viewer and the Send to Comparator button in the JSON Tracker both populate this page automatically via a browser session handoff — no copy-pasting required.

XML Comparator

Diff two XML documents.

Open tool →

Usage

Paste XML or load files into the two panels. Choose between order-sensitive (child element order is significant) and order-insensitive modes, then click Compare. The result lists structural differences — missing elements, differing attribute values, mismatched text content.

CSV Comparator

Find rows that are present in one CSV but not the other.

Open tool →

Usage

Paste or load two CSV files. Configure the delimiter (comma, semicolon, tab) and whether to skip the header row and blank lines. Click Compare. The result is a two-column table: rows only in File A, and rows only in File B. A preview limit caps the display for large files.

The comparison is multiset-based: if a row appears twice in A and once in B, one copy is considered matched and one is reported as extra.

Digital image processing

Photo Slicer

Cut an image into equal rectangular tiles and download them.

Open tool →

Usage

  1. Load a JPG, PNG, or TIFF image.
  2. Set the number of horizontal and vertical cuts.
  3. Optionally enable Force square to crop the source image to a square before slicing.
  4. Choose the alignment (9-point grid: top-left, center, bottom-right, etc.) which determines which part of the image is kept when forcing a square crop.
  5. Set the output format (JPG / PNG) and JPEG quality.
  6. Click Slice. A live preview is shown; click Download all to get all tiles as individual files.
